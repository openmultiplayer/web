## Description

- `config.json` 是一个服务端配置文件，允许你修改 open.mp 服务器的各种设置。
- 你依旧可以使用 [server.cfg](server.cfg) 在你的 open.mp 服务器中, 但是推荐使用 config.json!

<br />

:::tip

你可以很容易地将 server.cfg 转换成 config.json.

在你的服务器文件夹中打开命令行工具，然后输入下面的命令，按下回车键:

```bash
# Windows
omp-server --dump-config

# Linux
./omp-server --dump-config
```

或者生成一个默认的配置文件 config.json:

```bash
# Windows
omp-server --default-config

# Linux
./omp-server --default-config
```

:::

## Sleep and Ticks

| 键名          | 类型   | 默认值 | 只读 | 规则 | 作用                                                                                                                                                                                                                                                                                                                                                                                                        |
| ------------- | ------ | ------ | ---- | ---- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| sleep         | 浮点型 | 5.0    | ❌    | ❌    | 每个同步周期中，主要的 open.mp 和 raknet 网络线程将处于空闲状态的时间（毫秒）。较高的值会降低服务器处理能力，但会降低同步质量。较低的值会增加服务器处理能力，但会提高同步质量。除非您的玩家数量非常多且服务器 FPS 稳定性存在问题，否则不建议更改此值。                                                                                                                                                      |
| use_dyn_ticks | 布尔型 | true   | ✅    | ❌    | dynticks 配置基本上是为了通过更多地使用 CPU 来保持服务器的 tickrate 在一个恒定的计数。如果有任何掉帧，它会使用更多的 CPU 来弥补间隙。<br />它是使用提供的 sleep 值计算的，因此如果 sleep 为 5，则恒定的 tick 数为 1000 / 5 = 200 每秒。<br />open.mp 根据每个 tick 的代码执行时间动态修改内部的 sleep 值，以保持稳定的 200 tick，较低的 sleep 意味着更多的 CPU 使用（如果服务器代码编写良好，差异不会很大） |
