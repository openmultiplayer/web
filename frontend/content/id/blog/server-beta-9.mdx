Halo semuanya,

Kami baru saja merilis open.mp server beta 9, memperbaiki beberapa masalah yang dilaporkan dan menambahkan beberapa fitur baru. 
Perubahan yang paling mencolok adalah - dukungan SA:MP 0.3DL. 
Sekarang Anda dapat menjalankan server yang kompatibel dengan 0.3.7 dan 0.3.DL dari binary yang sama.

Perubahan Lainnya
-----------------

- Tingkat tick dinamis. Tentukan target tickrate, daripada waktu tidur yang tetap.
    * Gunakan perintah konsol tickrate untuk mengatur rate-nya.
    * sleep sekarang bisa berupa float untuk menyesuaikan tick rate yang tepat.
- Kompatibilitas plugin lebih banyak - fsutil, penghubung Discord (hanya dengan build non-statis)
- Memperbaiki pengembalian SetSpawnInfo.
- Memperbaiki cuaca dan waktu yang tidak reset setelah GMX.
- Memperbaiki berbagai bug GMX.
- Memperbaiki berbagai perintah konsol.
- Memperbaiki GetPlayerClass.
- Memperbaiki GetVehicleLastDriver.
- Menambahkan native GetVehicles.
- Menambahkan native GetPlayers.
- Menambahkan native GetActors.
- Semua mode lagcomp sekarang tersedia melalui konfigurasi.
- Memperbaiki sinkronisasi trailer.
- Memperbaiki GetVehicleLastDriver.
- Memperbaiki GetPlayerRotationQuat.
- Memperbaiki crash pada komponen kendaraan.

SA:MP DL
---------

Ini adalah berita besar tentunya, dan datang dengan banyak pengaturan baru:

- Mengimplementasikan semua native dan callback DL (AddSimpleModel, AddSimpleModelTimed, GetPlayerCustomSkin, OnPlayerFinishedDownloading, OnPlayerRequestDownload):
    * Native RedirectDownload dan callback OnPlayerRequestDownload sekarang sudah usang dan digantikan dengan opsi konfigurasi CDN.
- Menambahkan opsi konfigurasi artwork.enable (legacy: useartwork).
- Menambahkan opsi konfigurasi artwork.models_path (legacy: artpath).
    * Server kini bisa memuat model kustom dari file artconfig.txt seperti halnya SA:MP.
- Kompatibilitas 0.3.7 tetap dipertahankan dan bisa dikonfigurasi dengan opsi konfigurasi network.allow_037_clients (default true).
- Server open.mp sekarang dilengkapi dengan webserver bawaan yang digunakan untuk menyajikan model kustom.
    * Webserver menggunakan alamat bind dan port server open.mp. Harap izinkan koneksi TCP di port Anda di firewall jika Anda berencana menggunakannya.
    * Jika alamat IP publik Anda berbeda dari alamat bind (misalnya: Anda berada di belakang router), Anda perlu mengaturnya di opsi konfigurasi network.public_addr.

### Catatan

- Karena beberapa perubahan, file config.json lama mungkin tidak berfungsi dengan baik. Harap gunakan ./omp-server --default-config untuk menghasilkan yang baru.
- Pengguna server.cfg tidak terpengaruh.
- Sementara memungkinkan koneksi 0.3.7, hanya 1000 objek yang dapat dibuat.
- Server open.mp sekarang menyediakan dua jenis build Linux, standar dan statis. Disarankan untuk menggunakan build standar jika memungkinkan, karena akan meminimalkan ketidakcocokan dengan berbagai plugin; namun, ini akan memerlukan openssl 1.1 yang terpasang di sistem Anda. Jika Anda tidak bisa menginstal openssl modern di sistem Anda (biasanya jika sistem Anda sangat tua), Anda dapat menggunakan build statis, tetapi beberapa plugin atau komponen mungkin gagal dimuat dan menyebabkan server crash.

Official Includes
-----------------

Mempersiapkan rilis penuh tidak hanya tentang memiliki server yang bekerja, tetapi juga semua perangkat periferal lainnya - pustaka, alat, dokumentasi, dan lainnya. Di sisi pawn, open.mp adalah peningkatan signifikan, dengan banyak peningkatan QoL yang sudah terlihat melalui perbaikan dan fungsi baru, dan masih banyak lagi yang akan datang. Sebagian besar dari dorongan ini adalah lebih banyak diagnostik kompilator untuk menemukan masalah kode sebelumnya, terutama melalui lebih banyak tag dan keakuratan konstanta. Meskipun versi resmi belum dirilis, Anda tetap bisa membantu sementara dengan menguji kode Anda dengan includes dan alat berikut:

https://github.com/pawn-lang/samp-stdlib/tree/consistency-overhaul  
https://github.com/pawn-lang/pawn-stdlib/tree/backported-natives  
https://github.com/openmultiplayer/upgrade  
https://github.com/pawn-lang/compiler  

Ini awalnya dikembangkan untuk meningkatkan includes default SA:MP, tetapi tidak pernah sepenuhnya dirilis (yaitu, tidak pernah digabungkan ke master, meskipun PR telah diterima). Includes open.mp dibangun di atas dasar yang telah dibangun di sini, jadi anggaplah ini sebagai batu loncatan menuju keamanan tag dan konstanta penuh open.mp; dan umpan balik penting mengenai arah mereka. Lihat tautan di atas untuk informasi lebih lanjut dan dokumentasi tentang peningkatan.

Seperti biasa, jika Anda tidak ingin meningkatkan kode lama, includes masih akan bekerja (setidaknya untuk rilis pertama).
