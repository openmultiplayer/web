## 描述

- `config.json`是用于配置 open.mp 服务器各项参数的配置文件
- 你仍然可以在 open.mp 服务器中使用 [server.cfg](server.cfg) 文件，但建议使用 config.json，因为它有更多设置！

<br />

<hr />

<br />

:::tip

你可以简单地将你的 server.cfg 转换为 config.json。

在服务器目录中打开命令提示符，输入以下命令，然后按回车：

```bash
# Windows
omp-server --dump-config

# Linux
./omp-server --dump-config
```

**注意：** 如果目录中有 config.json 文件，你可能需要在执行上述命令前删除它。

:::

<br />

:::tip

要生成默认的 config.json，请输入以下命令：

```bash
# Windows
omp-server --default-config

# Linux
./omp-server --default-config
```

:::

| sleep         | float | 5.0    | ❌   | ❌   | 主 open.mp 和 raknet 网络线程在每个同步周期中将"休眠"的空闲时间（毫秒）。较高的值会减少服务器处理，但会降低同步质量。较低的值会增加服务器处理，但会提高同步质量。除非你的玩家数量非常高并且你有服务器 fps 稳定性问题，否则不建议更改此值。                                                                                                                                                        |
| use_dyn_ticks | bool  | true   | ✅   | ❌   | dynticks 配置基本上是为了通过使用更多 CPU 来覆盖任何间隙，将服务器的时钟周期保持在恒定计数。<br />它使用提供的 sleep 值计算，所以如果 sleep 是 5，恒定时钟周期数将是 1000 / 5 = 每秒 200 个时钟周期。<br />open.mp 根据每个时钟周期的代码执行时间动态修改内部 sleep 值，只是为了保持它在 200 个时钟周期稳定，较低的 sleep 意味着更多的 CPU 使用量（如果服务器代码编写良好，这不是一个巨大的差异） |
